# Connecting ScribeRAG to LM Studio (Local LLM Server)

This guide walks you through configuring **ScribeRAG** to connect to a local LLM instance running on **LM Studio** on your PC.

---

## 1. Configure LM Studio on your PC

By default, LM Studio only listens on `localhost` (`127.0.0.1`), which blocks requests from external devices like your mobile phone. You must enable local network sharing:

1. Open **LM Studio** on your PC.
2. Click the **Developer/Local Server** icon (the double-network-arrow icon) on the left sidebar.
3. Locate the **Local Server** settings panel:
   - In the settings, find the **Serve on Local Network** toggle.
   - Switch it **ON**. 
   - (Alternatively, set the **Host/Bind Address** configuration from `127.0.0.1` to `0.0.0.0` or select your Wi-Fi interface adapter IP address).
4. Load your desired model (e.g., `google/gemma-4-31b`).
5. Click **Start Server** if it is not already running.
6. The green status message at the top should now show that the server is reachable via a local IP (e.g., `http://192.168.X.X:1234` or `http://172.X.X.X:1234`).

---

## 2. Obtain your PC's Wi-Fi IP Address

Your PC might show virtual IP addresses (like `172.28.128.1`) which are generated by WSL2, Hyper-V, or Docker. Your phone **cannot** reach these virtual IPs. You must find your physical Wi-Fi IP address:

1. Open **Command Prompt** (`cmd`) on your Windows PC.
2. Run the command:
   ```cmd
   ipconfig
   ```
3. Locate the **Wireless LAN adapter Wi-Fi** section.
4. Note down your **IPv4 Address** (typically looks like `192.168.0.X` or `192.168.1.X`). 
   *(In your case, it was `192.168.0.142`)*.

---

## 3. Configure ScribeRAG Settings on your Phone

1. Ensure your phone is connected to the **same Wi-Fi network** as your computer.
2. Open ScribeRAG and go to the **Settings** screen.
3. Choose **LM Studio (Offline Local LLM)** from the **Select Provider** dropdown.
4. Input the connection configurations:
   - **Base URL:** Format it as `http://<YOUR-PC-WIFI-IP>:1234/v1`
     *(For example: `http://192.168.0.142:1234/v1`)*
   - **Model Name / ID:** Enter the exact model name currently loaded in LM Studio.
     *(For example: `google/gemma-4-31b`)*
5. Tap **Test Connection**. A **green checkmark** will appear once successfully connected.
6. Tap **Save Configuration**.

---

## 4. Troubleshooting & Firewall Setup

If the **Test Connection** fails:

* **Check the same Wi-Fi Subnet:** Verify that both the phone and PC share the exact same gateway/subnet (e.g. `192.168.0.X`).
* **Windows Defender Firewall Block:** 
  1. Open Windows Search and type **Allow an app through Windows Firewall**.
  2. Click **Change Settings** at the top right.
  3. Scroll down to find **LM Studio** and check the checkboxes for both **Private** and **Public** networks.
  4. Tap **OK** to save.
